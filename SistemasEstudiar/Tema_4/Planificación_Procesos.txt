1.- FIFO (First IN First Out)
    Los procesos se cargan según el orden de llegada.

2.- SJF (Sort Job First)
    Los procesos se ordenan por la cantidad de tiempo que van a usar la CPU.
    Se ordenan de menor a mayor.
    -> Entrarán primero los que menos vayan a usar la CPU. 
    Hasta que no termina un proceso no entra el otro.

Una vez que entran no salen hasta que terminan.


3.- RR (Round Robin)
    El más justo, puesto que reparte el mismo tiempo a todos.
    Su principal desventaja es que si los procesos no se terminan en el tiempo que se le ha otorgado, volverían al estado Listo, por lo que, 
    puede provocar muchos cambios de contexto.

4.- SRTF (Short Remain Time First)
    Variante el SJF. 
    Siempre va a estar dentro el que menos tiempo va a consumir.
    A diferencia del SJF, donde el proceso una vez ejecutado llegaba hasta su finalización, en este tipo de planificación de procesos se 
    realiza una evaluación cada vez que hay un proceso por entrar.
    Si el proceso por entrar tiene menos tiempo que el que está ejecutándose en ese momento, se le saca y se introduce el proceso que tiene
     menos tiempo de ejecución.
    Desventaja: si se unen muchos procesos con menos tiempo de ejecución que el que está ejecutándose, se pueden producir muchos cambios de contexto.

5.- Prioridades
    Los procesos entran según la prioridad que le asigna el SO. 
    Si entra un proceso con una prioridad más alta mientras se está ejecutando un proceso, el SO puede decidir si le expulsa 
    para que entre el nuevo con mayor prioridad.
