Las políticas de reemplazo se llevan a cabo en los intercambios de páginas de memoria principal y memoria secundaria.

Cuando la RAM está llena, el SO debe decidir cuál es la página, que estña cargada en la memoria principal, 
debe salir y ser llevada a la memoria secundaria para dejar marcos libres para nuevas páginas.


    1.- FIFO (First In First Out)
        El SO mantiene una lista de las páginas en memoria, siendo la página de cabeza de la lista la más antigua y la última de la lista, 
        la última en llegar. Ante una falta de página, se reemplaza la página de cabeza y la nueva página se añade al final de la lista.

        * Inconveniente: la página más vieja puede ser muy usada y aún así saldrá para tener que volver a entrar.

    2.- LRU (No usada recientemente)
        Requiere de un contador de software asociado con cada página e inicializado a 0. Este contador contiene las referencias a las páginas. 
        Cada vez que se referencia una página o se usa se incrementa el contador a 1. Cuando ocurre una falta de página, la página con el contador más bajo  es reemplazada.

    3.- NRU (No usada frecuentemente)
        A cada página le asociamos 2 bits: 
        
        - R: bit de referencia se pone a 1 en cualquier lectura o escritura en la página. 
        
       - M: bit de modificado se pone a 1 cuando se escribe en una página. 
       
        Cuando un proceso empieza, el sistema operativo coloca a 0 esos 2 bits (para todas las páginas). 
        Periódicamente (cada tick de reloj, que suele producirse a intervalos de 20 ms aproximadamente), el bit R se limpia
        (se pone a 0) para distinguir páginas no referenciadas recientemente de las que si lo han sido. Para sacar una página, 
        el sistema operativo examina los bits R y M de todas las páginas y saca una página al azar de la clase más baja no vacía:

    Clase 0: no referenciada (R=0), no modificada (M=0)
    Clase 1: no referenciada (R=0), modificada (M=1)
    Clase 2: referenciada (R=1), no modificada (M=0)
    Clase 3: referenciada (R=1), modificada (M=1) MEMORIA VIRTUAL.
